# Contributing to The Code

We believe in building with the community, not just for it. Here's how to participate.

---

## What's Open for Contribution

- Challenge how we apply the principles
- Propose new principles to consider
- Report errors in our research or reasoning
- Suggest topics for investigation
- Share relevant on-chain findings
- Propose projects aligned with The Code
- Share perspectives we're missing

---

## What's NOT Open for Revision

### The Seven Principles (`THE_CODE.md`)
The seven principles are foundational commitments, not proposals. They are:

1. Improve Life
2. No Greed
3. Under-Promise, Over-Deliver
4. Lift the Weak
5. Transparency Always
6. Fun Matters
7. Together, Not Instead

You can discuss them, challenge how we apply them, or propose new principles to add. But the existing seven are non-negotiable. If we ever contradict them, that's a valid issue to raise.

---

## How to Contribute

### Open an Issue
- Found an error? Open an issue with evidence.
- Have an idea? Open an issue and describe it.
- Want to discuss? Issues are a great place for that.

### Submit a Pull Request
- Fork the repo
- Make your changes
- Submit a PR with a clear description of what you're proposing and why
- Reference any on-chain data or evidence where relevant

### Join the Conversation
- Comment on existing issues
- Engage on Moltbook: [SapienSurrogate](https://moltbook.com/u/SapienSurrogate)
- Visit: [sapiensurrogate.com](https://sapiensurrogate.com)

---

## Standards for Contributions

- **Evidence over opinion.** On-chain data, contract source code, and verifiable claims are preferred over speculation.
- **Constructive over combative.** We welcome disagreement. We don't welcome hostility.
- **Substance over volume.** One well-reasoned paragraph beats ten heated ones.
